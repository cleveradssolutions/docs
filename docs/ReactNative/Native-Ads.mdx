---
title: Native Ads
description: This guide explains how to load native ads in a React Native app.
---

Native ads are ad assets presented to users using UI components that match your app’s look and feel.
In React Native, a loaded native ad is represented by `NativeAdContent`.

```ts
import {
  NativeAdLoader,
  NativeAdContent,
  AdChoicesPlacement,
} from 'react-native-cas';
```

## Add ad event listeners

Subscribe to loader events before loading ads.

```tsx
import { useEffect } from 'react';
import {
  NativeAdLoader,
  NativeAdContent,
  AdError,
  AdContentInfo,
} from 'react-native-cas';

useEffect(() => {
  const unsubLoaded = NativeAdLoader.addAdLoadedEventListener((ad: NativeAdContent) => {
    // Called when an ad is successfully loaded.
    console.log('Native Ad loaded');
  });

  const unsubFailedToLoad = NativeAdLoader.addAdFailedToLoadEventListener((error: AdError) => {
    console.log('Native Ad failed to load', error);
  });

  const unsubImpression = NativeAdLoader.addAdImpressionEventListener((info: AdContentInfo) => {
    console.log('Native Ad impression', info);
  });

  const unsubClicked = NativeAdLoader.addAdClickedEventListener(() => {
    console.log('Native Ad clicked');
  });

  const unsubFailedToShow = NativeAdLoader.addAdFailedToShowEventListener((error: AdError) => {
    console.log('Native Ad failed to show', error);
  });

  return () => {
    unsubLoaded();
    unsubFailedToLoad();
    unsubImpression();
    unsubClicked();
    unsubFailedToShow();
  };
}, []);
```

## Load ad content

Configure optional settings and request one native ad.

```ts
NativeAdLoader.setStartVideoMuted(true);
NativeAdLoader.setAdChoicesPlacement(AdChoicesPlacement.TOP_RIGHT_CORNER);
```

```ts
NativeAdLoader.loadAds(1);
```

A successfully loaded ad is delivered via `addAdLoadedEventListener`.

```ts
import { useState } from 'react';
import { NativeAdContent } from 'react-native-cas';

const [nativeAd, setNativeAd] = useState<NativeAdContent | null>(null);

NativeAdLoader.addAdLoadedEventListener((ad: NativeAdContent) => {
  setNativeAd(prev => {
    prev?.destroyAd(); // optional: replace previous ad
    return ad;
  });
});
```

## Load multiple ads (optional)

Request multiple native ads by passing a larger number to `loadAds`.
Each loaded ad will be delivered through `addAdLoadedEventListener` (potentially multiple times).

```ts
NativeAdLoader.loadAds(3);
```

One simple approach is to keep a small queue:

```ts
import { useState } from 'react';
import { NativeAdContent } from 'react-native-cas';

const [queue, setQueue] = useState<NativeAdContent[]>([]);

NativeAdLoader.addAdLoadedEventListener((ad: NativeAdContent) => {
  setQueue(prev => [...prev, ad]);
});
```

## Display ad content

Native ad content is displayed using `NativeAdView`. You have two approaches:

- **Template view** — quickest way to display native ads (no children inside `NativeAdView`).
  Continue with: [Native Ad templates](ReactNative/Native-Ads-Templates)

- **Manual asset views** — fully custom layout using asset components inside `NativeAdView`.
  Continue with: [Native Ad asset views](ReactNative/Native-Ad-Assets)

## Release ad resources

When an ad is no longer used, release resources by calling `destroyAd()`.

```ts
nativeAd?.destroyAd();
```

If you keep multiple ads (queue), release each one:

```ts
queue.forEach(ad => ad.destroyAd());
```

<Warning>
Always call `destroyAd()` when a loaded ad is no longer displayed.  
Do not display the same `NativeAdContent` in multiple views at the same time.
</Warning>

<Info>
Read more about `AdContentInfo` structure in [Impression Level Data](ReactNative/Impression-Level-Data).
</Info>
