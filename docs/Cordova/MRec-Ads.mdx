---
title: Medium Rectangle Banner Ads
description: This guide explains how to integrate medium rectangle banner ads into a Cordova app.
---

Medium Rectangle (MREC) are rectangular image or text ads that occupy a spot with size in dp (300x250) on screen. 
They stay on screen while users are interacting with the app, and can refresh automatically after a certain period of time. 

CAS provides an option to enable `autoReload` to simplify your app logic. The `autoReload` is enabled by default, loading failures will trigger automatic retry attempts.
You should decide whether to enable `autoReload`, or manually manage each ad loading cycle in your code.

Also, banner ads support automatic `refreshInterval` regardless of the `autoReload` setting.

## Load an MREC Ad
Call manual load before each show if Autoload Ads is not enabled in settings.

<Tabs groupId="style">
  <TabItem value="promise" label="Promise">
```js
// Using .then() / .catch()
casai.loadMRecAd({
  autoReload: true,
  refreshInterval: 30
})
.then(() => {
  console.log('MREC loaded');
})
.catch((error) => {
  console.error('MREC failed to load:', error);
});
```
  </TabItem>
  <TabItem value="async" title="Async"> 
```js 
// Using async/await 
async function loadMREC() { 
  try { 
    await casai.loadMRecAd({ 
      autoReload: true, 
      refreshInterval: 30 
    }); 
    console.log('MREC loaded'); 
  } 
  catch (error) { 
    console.error('MREC failed to load:', error); 
  } 
} 
``` 
  </TabItem> 
  <TabItem value="events" title="Events"> 
```js
// Events-driven approach (call load without awaiting, handle events separately) 
function onMrecEvent(e) { 
  const detail = e?.detail || {}; 
  switch (detail.type) { 
    case 'casai_ad_loaded': 
      console.log('MREC loaded (event)'); 
      break; 
    case 'casai_ad_load_failed': 
      console.error('MREC laod failed (event):', detail.error); 
      break; 
    case 'casai_ad_clicked': 
      console.log('MREC clicked (event)'); 
      break; 
    default: 
      console.log('MREC event:', detail); 
    } 
}

// Subscribe to banner events
document.addEventListener('casai_ad_loaded', onMrecEvent);
document.addEventListener('casai_ad_load_failed', onMrecEvent);
document.addEventListener('casai_ad_showed', onMrecEvent);
document.addEventListener('casai_ad_clicked', onMrecEvent);

// Trigger load (fire-and-forget; result handled via events)
casai.loadMRecAd({  
  autoReload: true,
  refreshInterval: 30
});

// To stop listening:
document.removeEventListener('casai_ad_loaded', onMrecEvent);
```
  </TabItem>
</Tabs>

## Display the MREC Ad
The position where the MREC ad should be placed. By default, it is centered in the screen.  
Changing the position **does not destroy** the current MREC ad or load a new one.

```js
// Show MREC (position is optional; default is CENTER)
casai.showMRecAd({
  position: casai.Position.CENTER
});

// Hide MREC
casai.hideMRecAd();
```

<Info> You can change the MREC's activity even if it is not ready to be shown yet. When the ad becomes ready, it will be shown automatically if autoload/show logic requires it. </Info>

## Release MREC ad resource
Destroys the ad content and releases any associated resources when the ad is no longer needed to prevent memory leaks.
```js
casai.destroyMRecAd();
```

Call `loadMRecAd` again to create and load a new ad instance and restore autoload behavior if needed.
