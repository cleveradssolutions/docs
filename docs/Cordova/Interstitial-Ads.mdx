Interstitial ads are full-screen ads that cover the interface of their host app.
They're typically displayed at natural transition points in the flow of an app, such as between screens.
When an app shows an interstitial ad, the user has the choice to either tap on the ad and continue to its destination or close it and return to the app.

Interstitial Ad management in React code is handled through the `InterstitialAd` class which extends the shared fullscreen ad logic.
The interstitial ad is displayed as an overlay on top of all app content and cannot be added to the Cordova component tree.

## Load Ad
CAS provides an option to enable autoload to simplify your app logic. 
Before using the ad, you should decide whether to enable autoload, or manually manage each ad loading cycle in your code.

When autoload is enabled, a new ad will be automatically loaded after each display, and loading failures will trigger automatic retry attempts.

<Tabs groupId="cordova-interstitial-load-autoload">
  <TabItem value="promise" label="Promise">

```js
casai.interstitialAd
  .load({
    autoReload: true,
    autoShow: false,
    minInterval: 0,
  })
  .then(function () {
    // Interstitial loaded with autoload enabled
  })
  .catch(function (error) {
    // Handle load error
  });
````

  </TabItem>
  <TabItem value="async" label="Async/Await">

```js
async function loadInterstitialAutoload() {
  try {
    await casai.interstitialAd.load({
      autoReload: true,
      autoShow: false,
      minInterval: 0,
    });
    // Interstitial loaded with autoload enabled
  } catch (error) {
    // Handle load error
  }
}
```

  </TabItem>
  <TabItem value="events" label="Events">

```js
casai.interstitialAd.load({
  autoReload: true,
  autoShow: false,
  minInterval: 0,
});
```

  </TabItem>
</Tabs>

Ad content may take some time to load after creating an ad instance.
To ensure the ad displays instantly when needed, load the ad instance in advance.

Use the `addAdLoadedEventListener` event to be notified when the ad is ready for display.

## Ad content callbacks

`InterstitialAd` has several optional callbacks for the ad lifecycle. These callbacks can be registered via event listeners.

```js
function isInterstitial(detail) {
  return detail && detail.format === 'Interstitial';
}

function onAdLoaded(e) {
  if (!isInterstitial(e.detail)) return;
  console.log('Loaded');
}

function onAdLoadFailed(e) {
  if (!isInterstitial(e.detail)) return;
  console.log('Load failed:', e.detail);
}

function onAdFailedToShow(e) {
  if (!isInterstitial(e.detail)) return;
  console.log('Failed to show:', e.detail);
}

function onAdShowed(e) {
  if (!isInterstitial(e.detail)) return;
  console.log('Showed');
}

function onAdClicked(e) {
  if (!isInterstitial(e.detail)) return;
  console.log('Clicked');
}

function onAdImpression(e) {
  if (!isInterstitial(e.detail)) return;
  console.log('Impression', e.detail);
}

function onAdDismissed(e) {
  if (!isInterstitial(e.detail)) return;
  console.log('Dismissed');
}

document.addEventListener('casai_ad_loaded', onAdLoaded);
document.addEventListener('casai_ad_load_failed', onAdLoadFailed);
document.addEventListener('casai_ad_show_failed', onAdFailedToShow);
document.addEventListener('casai_ad_showed', onAdShowed);
document.addEventListener('casai_ad_clicked', onAdClicked);
document.addEventListener('casai_ad_impressions', onAdImpression);
document.addEventListener('casai_ad_dismissed', onAdDismissed);

function unsubscribeInterstitialEvents() {
  document.removeEventListener('casai_ad_loaded', onAdLoaded);
  document.removeEventListener('casai_ad_load_failed', onAdLoadFailed);
  document.removeEventListener('casai_ad_show_failed', onAdFailedToShow);
  document.removeEventListener('casai_ad_showed', onAdShowed);
  document.removeEventListener('casai_ad_clicked', onAdClicked);
  document.removeEventListener('casai_ad_impressions', onAdImpression);
  document.removeEventListener('casai_ad_dismissed', onAdDismissed);
}
```

<Info>
- Read more about `AdContentInfo` structure in [Impression Level Data](Cordova/Impression-Level-Data).  
- When an error occurs during ad impression, only `AdFailedToShowEvent` is triggered.  
  In this case, `AdDismissedEvent` will not be executed since the impression is not considered successful.
</Info>

## Show Ad

Interstitial ads should be displayed during natural pauses in the flow of an app. Between app screens is a good example, or after the user completes a task.

You can choose when to show the ad by calling:

<Tabs groupId="cordova-interstitial-show">
  <TabItem value="promise" label="Promise">

```js
casai.interstitialAd
  .show()
  .then(function () {
    // Interstitial closed
  })
  .catch(function (error) {
    // Handle show error
  });
```

  </TabItem>
  <TabItem value="async" label="Async/Await">

```js
async function showInterstitial() {
  try {
    await casai.interstitialAd.show();
    // Interstitial closed
  } catch (error) {
    // Handle show error
  }
}
```

  </TabItem>
  <TabItem value="events" label="Events">

```js
casai.interstitialAd.show();
```

  </TabItem>
</Tabs>

If the ad is not ready at that moment, the `AdFailedToShowEvent` will be triggered with a reason.

You have the option to check `isLoaded()` if it's required by your app's logic.

```js
    const isAdLoaded = await casai.interstitialAd.isLoaded();
    // Use isAdLoaded boolean here
  } catch (error) {
    // Handle error (if your wrapper rejects)
  }
}
```

  </TabItem>
  <TabItem value="events" label="Events">

```js
var interstitialIsLoaded = false;

document.addEventListener('casai_ad_loaded', function (e) {
  if (e.detail && e.detail.format === 'Interstitial') {
    interstitialIsLoaded = true;
  }
});

document.addEventListener('casai_ad_dismissed', function (e) {
  if (e.detail && e.detail.format === 'Interstitial') {
    interstitialIsLoaded = false;
  }
});

if (interstitialIsLoaded) {
  casai.interstitialAd.show();
}
```

## Minimum interval between impressions

The minimum interval between showing interstitial ads, in seconds.
Attempting to show a new interstitial ad before the interval has passed after the previous one was dismissed will trigger `AdFailedToShowEvent` with `AdErrorCode.NOT_PASSED_INTERVAL`.

<Tabs groupId="cordova-interstitial-interval">
  <TabItem value="promise" label="Promise">

```js
casai.interstitialAd
  .load({
    autoReload: false,
    autoShow: false,
    minInterval: 30, // [!code highlight]
  })
  .then(function () {
    // Interval configured
  })
  .catch(function (error) {
    // Handle load error
  });
```

  </TabItem>
  <TabItem value="async" label="Async/Await">

```js
async function loadInterstitialWithInterval() {
  try {
    await casai.interstitialAd.load({
      autoReload: false,
      autoShow: false,
      minInterval: 30, // [!code highlight]
    });
    // Interval configured
  } catch (error) {
    // Handle load error
  }
}
```

  </TabItem>
  <TabItem value="events" label="Events">

```js
casai.interstitialAd.load({
  autoReload: false,
  autoShow: false,
  minInterval: 30, // [!code highlight]
});
```

  </TabItem>
</Tabs>

Note that the timer for the minimum interval is shared across all interstitial ad instances, but the minimum interval value may differ for each ad instance.

If you need to reset the minimum interval timer after showing a Rewarded Ad or an AppOpen Ad, you can call the `InterstitialAd.restartInterval()` method in the dismissal flow of those ad formats.

## Some best practices

#### Consider whether interstitial ads are the right type of ad for your app.

Interstitial ads work best in apps with natural transition points. The conclusion of a task within an app, like sharing an image or completing a game level, creates such a point. Because the user is expecting a break in the action, it's easy to present an interstitial ad without disrupting their experience. Make sure you consider at which points in your app's workflow you'll display interstitial ads and how the user is likely to respond.

#### Remember to pause the action when displaying an interstitial ad.

There are a number of different types of interstitial ads: text, image, video, and more. It's important to make sure that when your app displays an interstitial ad, it also suspends its use of some resources to allow the ad to take advantage of them. For example, when you make the call to display an interstitial ad, be sure to pause any audio output being produced by your app.

#### Allow for adequate loading time.

Just as it's important to make sure you display interstitial ads at an appropriate time, it's also important to make sure the user doesn't have to wait for them to load. Loading the ad in advance by calling `loadAd()` before you intend to call `showAd()` can ensure that your app has a fully loaded interstitial ad at the ready when the time comes to display one.

#### Don't flood the user with ads.

While increasing the frequency of interstitial ads in your app might seem like a great way to increase revenue, it can also degrade the user experience and lower clickthrough rates. Make sure that users aren't so frequently interrupted that they're no longer able to enjoy the use of your app.
